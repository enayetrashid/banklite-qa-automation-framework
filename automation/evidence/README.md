# Evidence and Logs

This folder stores test execution evidence generated by the automation framework.

## Subfolders
- `logs/` stores:
  - pytest execution log output
  - request and response evidence for API tests
- `screenshots/` is reserved for browser evidence from future or extended UI automation

## What gets generated
API tests produce JSON evidence files containing:
- request method
- request URL
- request payload
- response status code
- response body
- UTC capture timestamp

Pytest also writes a consolidated execution log to:
- `automation/evidence/logs/test_execution.log`

## Recommended usage
- keep `.gitkeep` files committed
- do not commit large generated evidence files unless needed for a report or bug record
- attach selected evidence to bug reports and execution summaries
